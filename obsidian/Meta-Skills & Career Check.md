### 7.1 Engineering Culture

То, что отличает сеньора от джуна.
*   **Documentation:** README.md должен быть таким, чтобы незнакомец запустил проект за 5 минут. Docstrings в коде обязательны.
*   **Git Etiquette:** Атомарные коммиты (одна логическая задача — один коммит). Понятные сообщения (`feat: add user login`, `fix: memory leak in parser`).
*   **Debugging:** Умение пользоваться отладчиком (pdb/ipdb в Python, VS Code Debugger), а не только `print()`.
*   **Logging:** Вместо `print` использовать `logger.info/error`. Структурные логи (JSON).

### 7.2 System Design (Basics)
Как проектировать системы, которые не падают.
*   **Scalability:** Вертикальное vs Горизонтальное масштабирование. Load Balancers (Nginx).
*   **Caching:** Когда использовать Redis/Memcached. Проблемы инвалидации кэша.
*   **Queues:** Зачем нужны брокеры сообщений (RabbitMQ, Kafka, Redis Streams) для асинхронных задач.

### 7.3 Continuous Integration / Delivery (CI/CD)
Автоматизация рутины.
*   **GitHub Actions:** Написать пайплайн, который:
    1.  При пуше запускает линтер (Ruff/Flake8).
    2.  Запускает тесты (Pytest).
    3.  Собирает Docker образ (опционально).

## 8. Continuous Learning Loop

Технологии устаревают. План поддержки формы.

**Еженедельно:**
1.  **LeetCode:** 1 задача Medium уровня (поддержание формы мозга).
2.  **News:** Читать рассылки (например, Python Weekly, Hacker News), чтобы знать о новых версиях библиотек.

**Ежемесячно:**
1.  **Refactoring:** Открывать свой старый код (месячной давности) и ужасаться. Переписывать кусок с новыми знаниями.
2.  **Pet Project Review:** Обсуждать с напарником, что можно улучшить в текущем проекте (добавить кэш? ускорить базу? переписать на Rust?).
