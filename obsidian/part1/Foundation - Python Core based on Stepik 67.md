

Базовый модуль. Цель — закрыть пробелы в синтаксисе и алгоритмическом мышлении перед переходом к ООП и архитектуре.

**Источник программы:** Stepik Course 67 (Bioinformatics Institute).
**Методология:** Pair Programming + Code Review.

---

## 1. Операторы и Пространство имен

Базовая арифметика и логика. Работа с типами данных.

**Темы**
1.  Арифметика: деление, остаток, приоритет операций.
2.  Типы данных: `int`, `float`, `bool`. Особенности float (погрешность).
3.  Логические операции: `and`, `or`, `not`. Таблицы истинности.

**Задачи (Hard Mode)**
*Вместо простых вычислений пишем утилиты.*

*   **Задача 1.1: Калькулятор площадей**
    Реализовать программу, которая принимает тип фигуры (треугольник, прямоугольник, круг) и соответствующие параметры, а затем вычисляет площадь. Использовать формулу Герона для треугольника.
*   **Задача 1.2: Сортировка без массивов**
    Написать программу, которая принимает три числа и выводит их в порядке возрастания, а также отдельно минимальное и максимальное, используя только `if/else` (без списков и функций `min/max`).
*   **Задача 1.3: Склонение слов**
    Программа принимает число "программистов" и выводит правильное окончание (1 программист, 2 программиста, 5 программистов).

## 2. Циклы и Строки

Управление потоком выполнения. Обработка текста.

**Темы**
1.  Цикл `while`: условия выхода, бесконечные циклы.
2.  Цикл `for`: `range()`, итерация.
3.  Строки: методы `str`, срезы (slicing), неизменяемость строк.
4.  Алгоритм RLE (Run-Length Encoding).

**Задачи (Hard Mode)**
*   **Задача 2.1: RLE Сжатие (DNA Compression)**
    На вход подается строка вида `AAAABBBCCXYZDDDDEEEFFFAAAAAABBBBBBBBBBBBB`.
    Написать алгоритм, который сжимает её в вид `A4B3C2XYZD4E3F3A6B13`.
    *Усложнение:* Написать и декодер, возвращающий исходную строку.
*   **Задача 2.2: GC-состав**
    Для строки генома посчитать процентное соотношение символов G и C. (Работа с регистром символов).
*   **Задача 2.3: Сумма последовательности**
    Считывать числа с ввода до тех пор, пока их сумма не станет равна 0. После этого вывести сумму квадратов всех считанных чисел. Внимательно следить за потреблением памяти (не сохранять список, считать на лету).

## 3. Списки и Многомерные массивы

Работа с памятью и структурированными данными.

**Темы**
1.  Списки: методы `append`, `sort`, `insert`. Изменяемость списков.
2.  Цикл `for` по элементам vs по индексам.
3.  Генераторы списков (List Comprehensions).
4.  Двумерные массивы (матрицы).

**Задачи (Hard Mode)**
*   **Задача 3.1: Сапер (Minesweeper)**
    Поле заданного размера. На вход подаются координаты мин. Нужно вывести поле, где в каждой пустой клетке стоит число, означающее количество мин вокруг неё.
*   **Задача 3.2: Матричное умножение**
    Не используя `numpy`, реализовать умножение двух матриц произвольного размера. Добавить проверку на совместимость размерностей.
*   **Задача 3.3: Спиральная матрица (Final Boss)**
    Вывести матрицу размера `n * n`, заполненную числами от 1 до `n^2` по спирали (по часовой стрелке), начиная с левого верхнего угла.
    *Пример для n=3:*
    ```
    1 2 3
    8 9 4
    7 6 5
    ```

## 4. Функции, Словари и Файлы

Структурирование кода и работа с внешними данными.

**Темы**
1.  Функции: аргументы, `return`, локальные переменные.
2.  Словари (`dict`): методы, итерация по ключам/значениям. Частотный анализ.
3.  Файловый ввод/вывод: чтение, запись, режимы доступа.
4.  Модули: импорт `math`, `sys`. Установка внешних пакетов (`requests`).

**Задачи (Hard Mode)**
*   **Задача 4.1: Частотный анализ текста**
    Считать текст из файла. Подсчитать частоту каждого слова (регистронезависимо). Вывести самое часто встречающееся слово. Если таких несколько — вывести лексикографически первое.
*   **Задача 4.2: Средний балл**
    В файле список студентов и их оценки по трем предметам.
    Вывести:
    1. Средний балл каждого студента.
    2. Средний балл по каждому предмету среди всех студентов.
    Результат записать в новый файл.
*   **Задача 4.3: Расшифровка (Dataset Request)**
    С использованием библиотеки `requests`.
    Скачать файл по ссылке. В файле может быть текст или ссылка на другой файл. Если ссылка — скачать следующий. Пройти цепочку до конца, пока не найдется текст, начинающийся с кодового слова.

## Проектная работа по курсу

Объединение всех тем в одну утилиту.

**CLI Утилита: Анализатор Логов**
Написать консольную утилиту, которая:
1.  Принимает имя лог-файла (формат apache/nginx).
2.  Парсит файл построчно (не загружая весь в память).
3.  Считает статистику: количество запросов с каждого IP, самые популярные URL.
4.  Сохраняет отчет в CSV файл.
5.  Обрабатывает ошибки (файл не найден, битая строка).
